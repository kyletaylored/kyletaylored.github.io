OVERVIEW

Most agent frameworks feel like flying a plane with a blanket over the instruments. I built this one so you can see exactly what's happening at every step.

LLM Agent Framework is a minimal TypeScript library for building AI agents that use tools, maintain memory across sessions, and produce structured outputs. The design goal was legibility — every layer is thin enough to read and understand without a docs tab open.

It supports Claude, GPT-4, and any OpenAI-compatible endpoint out of the box. Tool definitions are plain TypeScript functions with Zod schemas. Memory is pluggable — in-memory for development, vector store for production.

FEATURES
◈
Tool use
Define tools as typed TS functions. Schema validation via Zod. Parallel calls supported.
◉
Memory layers
Short-term, long-term, and episodic. Plug in any vector store. Works offline with in-memory.
⬡
Structured output
Zod schema → guaranteed JSON. No more parsing surprises in production.
↻
Trace logging
Every think/tool/respond step logged with timing and token cost. OpenTelemetry compatible.
✕
Hard timeouts
Wall-clock kill switch built in. Agents can't run forever, and you'll know when they try.
</>
Multi-model
Claude, GPT-4, Mistral, Ollama. Swap models per-agent or mid-run.
QUICK START
agent.ts 
import { Agent, tool } from '@kyletaylor/agent';

const searchTool = tool({
  name: 'search_web',
  description: 'Search the internet for information',
  schema: z.object({ query: z.string() }),
  run: async ({ query }) => fetchResults(query),
});

const agent = new Agent({
  model: 'claude-opus-4',
  tools: [searchTool],
  memory: new VectorMemory(chromaClient),
  timeout: 30_000,
});

const result = await agent.run('Research the latest in RAG techniques');
// { output: '...', steps: 4, tokens: 1249, cost: 0.0089 }
WHAT I LEARNED
01
Context management is the real problem

Trimming context intelligently — not just truncating — is worth 10x the effort put into prompt engineering. Most agent failures are context failures.

02
Observability is non-negotiable

If you can't see the trace, you can't debug it. Instrument everything from day one — cost, latency, tool calls, token usage.

03
Smaller models + better tools beat bigger models every time

A well-defined tool schema with a fast model consistently outperformed a frontier model with vague tool descriptions.
